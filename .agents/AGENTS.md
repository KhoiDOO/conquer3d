# Agent Instructions

<RULE[user_global]>
If the user uses the `[COMMIT]` tag, follow these steps exactly:
1. Create a separate, individual commit for **each** changed file.
2. Write a detailed, descriptive commit message for each file.
3. Start every commit message with a conventional commit prefix (like `feat:`, `chore:`, `build:`, `refactor:`, `docs:`, etc.).
4. Ensure absolutely all changed files are committed.
5. Push the commits to GitHub immediately afterward.
</RULE[user_global]>
